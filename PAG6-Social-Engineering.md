# Concepto de Ingenieria Social

La ingeniería social es una técnica de ataque que manipula la psicología humana para obtener acceso no autorizado a información, sistemas o lugares físicos. En lugar de usar solo hacking técnico,
los atacantes engañan a las personas para que revelen datos confidenciales o realicen acciones inseguras. 
Incluye métodos como phishing, vishing, smishing, pretexting, baiting y tailgating. 
Muchos ataques implican suplantación de identidad, donde el atacante se hace pasar por alguien de confianza. La mejor defensa es la concienciación y formación en seguridad, para que los usuarios
puedan reconocer y evitar estos ataques. También existen campañas de influencia, fraudes y otras técnicas como shoulder surfing, dumpster diving o eavesdropping, que explotan errores humanos para comprometer la seguridad.

# Desencadenantes motivacionales

Los ingenieros sociales utilizan desencadenantes motivacionales para manipular a las personas y hacer que realicen acciones que comprometan la seguridad. Estos desencadenantes aprovechan la psicología humana para convencer a las víctimas.
Existen seis principales:

  - Autoridad: el atacante se hace pasar por alguien con poder (jefe, gobierno, cliente importante) para que la víctima obedezca.

  - Urgencia: crea presión de tiempo para que la persona actúe rápido sin pensar ni seguir los procedimientos de seguridad.

  - Prueba social: hace creer que muchas personas confían o usan algo, para que la víctima también lo haga.

  - Escasez: presenta una oferta o recurso como limitado para forzar una decisión rápida.

  - Simpatía: el atacante se muestra amable o agradable para ganarse la confianza de la víctima.

  - Miedo: usa amenazas o consecuencias negativas para obligar a la víctima a actuar.

Estos factores pueden usarse solos o combinados para hacer los ataques de ingeniería social más efectivos.

# Interpretación & Suplantación de Identidad

La suplantación de identidad es una técnica de ingeniería social en la que un atacante finge ser otra persona o entidad para obtener acceso a información o sistemas. 
Existen cuatro formas principales:

  - Suplantación (Impersonation): el atacante se hace pasar por un empleado, técnico de TI o persona con autoridad para ganar confianza y obtener datos o acceso.

  - Suplantación de marca (Brand impersonation): el atacante finge representar a una empresa/marca conocida usando su logotipo, correos o sitios web falsos para robar información.

  - Typosquatting: registro de dominios web muy parecidos a los reales pero con errores tipográficos para engañar a los usuarios y redirigirlos a sitios fraudulentos.

  - Watering hole: el atacante compromete un sitio web que la víctima suele visitar para infectar su sistema o lanzar ataques.

Comprender estas técnicas permite aplicar medidas de seguridad y formación para prevenir ataques de ingeniería social.

# Pretexting (Pretexto)

El pretexting es una técnica de ingeniería social en la que un atacante inventa una historia creíble (pretexto) para engañar a una persona y obtener información.

El atacante suele dar primero algunos datos que parecen reales para ganar confianza y luego pedir información adicional. En el ejemplo, el atacante llama a una empresa fingiendo ser de una compañía
de tóner de impresoras para obtener datos como el modelo de la impresora o su dirección IP, información que luego podría usar para futuros ataques.

Este tipo de ataques también ocurre en estafas telefónicas, como cuando alguien finge ser soporte técnico de una empresa y pide acceso al ordenador.
La mejor defensa es formar a los empleados para que no proporcionen información por teléfono o a desconocidos, incluso si parece información poco importante.

# Phishing

El phishing es una técnica de ingeniería social que busca engañar a las personas para que revelen información sensible (contraseñas, datos bancarios, etc.) haciéndose pasar por una entidad confiable. 
Existen varios tipos principales:

  - Phishing: envío masivo de correos falsos para engañar a muchas personas.

  - Spear phishing: ataque más dirigido y personalizado contra personas u organizaciones específicas.

  - Whaling: tipo de spear phishing dirigido a altos ejecutivos o personas importantes de una empresa.

  - Business Email Compromise (BEC): uso o suplantación de cuentas de correo empresariales para pedir transferencias de dinero o información.

  - Vishing: phishing realizado por teléfono para obtener datos confidenciales.

  - Smishing: phishing realizado mediante mensajes SMS con enlaces o solicitudes fraudulentas.

Conocer estos tipos de ataques permite identificarlos y aplicar medidas de seguridad para proteger a los usuarios y a las organizaciones.

# Prevención de Ataques de Phishing

Para prevenir ataques de phishing, las organizaciones deben combinar formación en seguridad, campañas antiphishing y detección temprana. Las campañas antiphishing enseñan a los usuarios a reconocer correos fraudulentos
y suelen incluir simulaciones de phishing para entrenar a los empleados. Los usuarios deben aprender a identificar señales comunes de phishing, como:

  - Urgencia: mensajes que presionan para actuar rápido.

  - Peticiones inusuales: solicitud de contraseñas o datos sensibles.

  - URL que no coinciden: el enlace visible no corresponde con la dirección real.

  - Direcciones de correo sospechosas: remitentes falsos o dominios extraños.

  - Errores de ortografía o gramática.

Si se detecta un correo sospechoso, debe reportarse al equipo de seguridad y evitar hacer clic en enlaces o compartir información.

En general, la mejor defensa contra el phishing es educación continua, vigilancia y respuesta rápida, lo que reduce significativamente el riesgo para las organizaciones.
